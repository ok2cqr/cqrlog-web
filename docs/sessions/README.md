# Session logs

Distilled summaries of substantial working sessions — continuity for the
next agent (or the next you), not transcripts.

## Convention

- One file per session: `YYYY-MM-DD-<kebab-topic>.md` (same topic twice a
  day → `-2`, `-3` suffix).
- Hard cap ~80 lines. Pointers + decisions + next actions, no play-by-play.
- Sections: Context, What was done, Key decisions, Open questions,
  Next steps, References.
- Existing logs are immutable — amend by writing a new dated file.
- Link commits/specs/ADRs by hash or path; never copy their content in.

Start a new session by reading the most recent 1–2 logs here.
